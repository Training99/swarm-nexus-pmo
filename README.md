# Swarm Nexus PMO - 蜂群项目管理中心 V0.1

## 🎯 核心职责

Swarm Nexus PMO (Project Management Office) 是管理所有蜂群的中心枢纽，24/7不间断运行：

| 职责 | 描述 | 频率 |
|------|------|------|
| **任务防遗漏** | 检测被遗忘的任务并自动重新分配 | 每5分钟 |
| **蜂群健康监控** | 监控所有蜂群状态，失联时自动恢复 | 每10分钟 |
| **安全监督** | 持续安全扫描，关键告警立即升级 | 每小时 |
| **每日站会** | 自动生成蜂群状态报告和协作需求 | 每天09:00 NY |
| **创新求解** | 当所有蜂群都无法解决问题时介入 | 按需 |

## 🧠 Innovation Agent

当蜂群遇到无法解决的问题时，Innovation Agent会介入并提供创造性解决方案：

**触发条件：**
- 任务没有匹配的蜂群能力
- 蜂群阻塞超过30分钟
- 所有尝试的方案都失败

**解决策略：**
1. **任务分解** - 将复杂任务拆分为简单子任务
2. **跨蜂群借调** - 组建联合工作组
3. **外部资源** - 搜索GitHub/文档/社区寻找方案
4. **快速学习** - 30分钟-2小时内掌握新能力
5. **降低范围** - 先交付MVP，后续迭代

```
┌─────────────────────────────────────────────────────────────────┐
│                    SWARM NEXUS SYNC BOARD                        │
├─────────────────────────────────────────────────────────────────┤

│ 🟡 opportunity_swarm     ← 状态指示 (🟢运行 🟡空闲 🔴阻塞)
│    状态: idle            ← 当前状态
│    队列: 0               ← 待处理任务数
│    性能: ██████████      ← 性能评分 (0-100%)
│    当前: 研究Anthropic   ← 正在执行的任务

├─────────────────────────────────────────────────────────────────┤
│ 📊 任务统计: 待分配 X | 进行中 X | 阻塞 X | 遗忘 X  ← 全局统计
└─────────────────────────────────────────────────────────────────┘
```

**状态指示：**
- 🟢 **active** - 蜂群正在执行任务
- 🟡 **idle** - 蜂群空闲，等待任务
- 🔴 **blocked** - 蜂群阻塞，需要帮助
- ⚪ **error** - 蜂群错误，需要恢复

## 🔄 任务生命周期

```
用户创建项目
    ↓
Nexus自动分解为任务
    ↓
任务状态: PENDING (待分配)
    ↓
Nexus根据能力匹配度自动分配 → ASSIGNED (已分配)
    ↓
蜂群开始执行 → IN_PROGRESS (进行中)
    ↓
执行完成 → COMPLETED (已完成)
    ↓
    ├─ 遇到问题 → BLOCKED (阻塞) → Innovation Agent介入
    └─ 超过2小时无进展 → ORPHANED (被遗忘) → 重新分配
```

## 🛡️ 安全监控

Nexus每小时执行一次安全扫描，检测：

| 告警级别 | 响应时间 | 处理方式 |

| 🟡 **MEDIUM** | 24小时内 | 日汇总报告 |
| 🟢 **LOW** | 记录 | 周汇总 |

## 🐛 故障排除

**Q: Nexus启动后没有输出**
A: 检查是否有global_memory.json损坏，删除后重启

**Q: 任务没有被分配**
A: 检查蜂群capabilities是否匹配任务required_capabilities

**Q: Innovation Agent没有触发**
A: Innovation Agent只在任务完全无法分配或蜂群阻塞时触发
